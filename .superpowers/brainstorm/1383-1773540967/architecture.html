This avoids CORS issues and enables request logging, caching, and error handling.

Key Design Decisions

P

Proxy Pattern

All Ollama API calls go through Next.js API routes. The browser never hits Ollama directly. This solves CORS, enables logging, and lets us add caching or rate limiting later.

S

SQLite for State

Settings (saved Ollama endpoints), pull history, and Modelfile drafts persist in a local SQLite database. Zero configuration, ships with the app.

E

Configurable Endpoint

Ollama server URL is a setting, not hardcoded. Defaults to localhost:11434 but can point to any Ollama instance (e.g., 192.168.50.4:11434 for your Orin).
